# MVC设计模式

# 什么是MVC设计模式

MVC是指：模型层Model、视图层View、控制层Controller。

  • Model：模型，负责对客户端提交请求进行处理，其包括实体类，Services 层，Dao层。实体类用于承载业务数据，Service层负责业务处理，Dao层用于负责操作数据库，获取结果数据，将结果数据集中封装到OV对象中，之后再返回给Service层。
  • View：视图，接收客户端请求并显示处理后的请求数据。
  • Controller：控制器，将用户请求转发给相应的Model进行处理，并处理Model的处理结果向用户提供相应响应。

# MVC设计模式工作流程

  1. 用户通过视图View页面向服务端提交请求
  2. 服务端Controller 控制器接收到请求后对请求解析，找到相应的Model对用户请求进行处理
  3. Model处理后，将处理结果再交给Controller。
  4. Controller在接到处理结果后，根据处理结果找到要作为向客户端发挥响应的View页面，页面经过渲染（数据填充）后，再发给客户端。

# MVC与三层架构的关系

MVC中的V和C对应三层架构中的View层，MVC中的M对应三层架构的中的Service层和DAO层

# SSM与三层架构的关系

SSM即SpringMVC、Spring、Mybaits三个框架，它们在三层架构中所处的位置是不同的。

  • SpringMVC：位于视图层，用于完成客户端请求的接收，转发及响应。

  • Spring：用于管理整个应用中所有对象的创建，初始化，销毁及对象间关联对象的维护。

  • Mybaits：位于DAO层，用于完成对数据库的增删改查等操作。
