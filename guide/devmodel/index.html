# TDD测试驱动开发

# 什么是TDD

TDD测试驱动开发是一种代码设计方法，也是一种简单的开发流程。 在开发功能代码之前，要求开发工程师先编写测试，再编写功能代码。

# 为什么要使用TDD

  • 可以改善设计和代码质量
  • 可简化重构工作
  • 可降低耦合度

# 测试驱动开发流程

  1. 编写一个测试
  2. 运行所有测试
  3. 编写实现代码
  4. 运行所有测试
  5. 重构
  6. 运行所有测试
