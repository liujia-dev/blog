# 单例模式

# 什么是单例模式

单例模式是一种软件设计模式，它的核心就是单例类。 所谓单例类就是一个仅能创建一个实例的类。

# 适用场景

  1. 资源共享的情况下，避免由于资源操作时导致的性能或损耗等。如日志文件，应用配置等。
  2. 控制资源的情况下，方便资源之间的互相通信。如线程池等。

# 单例实现方式

单例实现方式有饿汉式和懒汉式。

# 饿汉式

package com.tms;

/**
 * @author ：liu jia
 * @description：饿汉式单例演示程序
 * @date ：2021/4/30 20:24
 */
class Animal{
    //1、私有化类的构造器
    private Animal(){}
    //2、创建类的静态对象
    private static Animal instance = new Animal();
    //3、提供公共的静态方法, 返回类的对象
    public static Animal getInstance(){
        return instance;
    }
}
public class SingletonTest1 {
    public static void main(String[] args) {
        Animal animal1 = Animal.getInstance();
        Animal animal2 = Animal.getInstance();
        if(animal1==animal2){
            System.out.println("饿汉式单例类创建成功！");
        }
    }
}

# 懒汉式

package com.tms;

/**
 * @author ：liu jia
 * @description：懒汉式单例演示程序
 * @date ：2021/4/30 20:24
 */
class Animal{
    //1、私有化类的构造器
    private Animal(){}
    //2、声明当前类的静态对象
    private static Animal instance = null;
    //3、提供公共的静态方法, 返回当前类对象的方法
    public static synchronized Animal getInstance(){
        if(instance == null){
            instance = new Animal();
        }
        return instance;
    }
}
public class SingletonTest1 {
    public static void main(String[] args) {
        Animal animal1 = Animal.getInstance();
        Animal animal2 = Animal.getInstance();
        if(animal1==animal2){
            System.out.println("懒汉式单例类创建成功！");
        }
    }
}

# 饿汉式和懒汉式区别？

饿汉式：直接创建类的实例化，效率较高。 懒汉式：需要使用的时候再创建类的实例化，效率较低。
