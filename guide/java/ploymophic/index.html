# 多态

多态是面向对象的重要特征之一。

# 多态的三个必要条件

  • 继承
  • 重写
  • 向上转型

# 什么是继承

继承就是子类可以从父类继承得到成员变量和方法。

# 什么是重写

重写就是在子类中编写与父类中的方法名和参数都一样的方法。

# 什么是向上转型

所谓向上转型就是一个子类对象直接赋给一个父类引用变量，向上转型由系统自动完成。

向上转型格式：BaseClass baseClass = new SubClass();

# Java多态的演示程序

package com.tms;

/**
 * @author ：liu jia
 * @description：Java多态的演示程序
 */
class Animal {
    public void eat() {
        System.out.println("动物会吃东西");
    }
    public void sleep() {
        System.out.println("动物会睡觉");
    }
}
public class Dog extends Animal {

//   重写父类Animal中的eat()方法。
    public void eat() {
        System.out.println("小狗正在啃骨头");
    }

    public void play() {
        System.out.println("小狗正咬着布娃娃玩呢");
    }

    public static void main(String[] args) {
//      向上转型：一个子类Dog对象直接赋给一个父类Animal引用变量animal
        Animal animal = new Dog();
        animal.eat();
        animal.sleep();
        //animal.play();
        ((Dog) animal).play();
    }
}

# 运行

小狗正在啃骨头
动物会睡觉

# 为什么执行animal.eat()时，调用的是子类Dog的eat()方法，而不是父类Animal的eat()方法？

因为引用变量animal编译时类型是Animal，而运行时类型是Dog， 所以当调用引用变量animal的eat()方法时，实际执行的是Dog类中的eat()方法。

# 注释了animal.play();，这行代码为什么在编译时引发错误？

因为引用变量animal只能调用它编译时类型Animal的方法，而不能调用它运行时类型Dog的方法， 即使引用变量animal确实包含play()方法，所以编译时无法调用play()方法。

# 如何解决animal.play();代码在编译时引发的错误?

TIP

解决方案：((Dog)animal).play();

因为只有编译时类型和运行时类型完全相同，引用变量animal才能正常调用它们的成员变量和方法。 所以必须把引用变量animal运行时类型强制转换为Dog，这样才能调用play()方法。
