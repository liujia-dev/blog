# 接口

接口定义了一种规范，体现了规范和实现分离的哲学思想。

# 接口定义

接口定义的基本语法: [修饰符] interface 接口名 extends 父接口1，父接口2...

  • 接口中定义的成员变量只能是常量。
  • 接口中定义的普通方法只能是public 的抽象方法。
  • 接口中定义的默认方法，需要使用default修饰。
  • 接口中定义的类方法，需要static 修饰。

# 接口和抽象类

# 相同点

  • 接口和抽象都不能被实例化。
  • 接口和抽象类都可以包含抽象方法，实现接口或继承抽象类的普通子类都必须实现这些抽象方法。

# 不同点

  • 设计目的不同，接口体现的是一种规范，抽象类提现的是一种模板式设计。
  • 接口中不能为普通方法提供方法实现但抽象类则可以包含普通方法。
  • 接口中不能定义普通成员变量但抽象类中可以定义普通成员变量。
  • 接口中不包含构造器但抽象类可以包含构造器。
  • 接口中不能包含初始化块但抽象类可以包含初始化块。
  • 接口中可以继承多个接口但抽象类只能继承一个父类。

# 如何使用接口和实现类

接口名称 引用名 = new 实现类名称();

注意：调用的时候只能调用接口中定义的方法，运行时只能运行实现类中具体方法的内容

# 接口演示程序

# 定义接口 IAnimal

package com.tms;

/**
 * @author ：liu jia
 * @description：定义接口IAnimal
 * @date ：2021/5/1 18:17
 */
public interface IAnimal {
    void eat();
    void sleep();
}

# 定义IAnimal接口的实现类 CatImpl

package com.tms;

/**
 * @author ：liu jia
 * @description：定义接口IAnimal实现类CatImpl
 * @date ：2021/5/1 18:30
 */
public class CatImpl implements IAnimal {
 public void eat() {
  System.out.println("猫吃鱼");
 }

 public void sleep() {
  System.out.println("猫睡觉");
 }
 public void work() {
  System.out.println("猫抓鼠");
 }
}

# 使用接口实现类


package com.tms;

/**
 * @author ：liu jia
 * @description：使用接口实现类
 * @date ：2021/5/1 19:13
 */
public class Test {
 public static void main(String[] args) {
  IAnimal cat  = new CatImpl();
  cat.eat();
  cat.sleep();
  }
}
